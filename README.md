## История

1981 год. Nintendo of America находилась на грани краха, и тогда Сигэру Миямото 
разработал игру о противостоянии гориллы и маленького прыгающего человечка. 
Это была знаменитая Donkey Kong, которая положила начало карьере пухлого водопроводчика 
по имени Марио. С тех пор он появился более чем в двух сотнях игр, принес компании 
миллиардную прибыль и стал более узнаваемым, чем Микки-Маус.  



## Принципы, на основе которых написана игра?

1. Логика независима от визуального представления. Применяется архитектурный принцип **MVC**.

2. Приложение **кроссплатформенное**: логика не зависит от ОС.

3. Логика работает не с множествами объектов, а с множествами абстракций.

4. Для написания логики игры используется **полиморфизм**.

5. В игре применяются паттерны проектирования: 
	**Singleton**, **Factory**, **Adapter**, **Iterator**.


## Управление клавиатурой

В консоли windows (cmd) через `#include <windows.h>` можно читать состояние клавиш. 
Терминал linux работает по старым принципам телетайпа 1960-х гг. Он был создан для работы 
с последовательным портом, где нет понятия "клавиша нажата/отжата" — есть только поток символов. 
Поэтому для linux я решил изменить механику управления:

- Windows:
	- `A` - движение влево
	- `D` - движение вправо
	- `Space` - прыжок
	- `Q` - выход из программы
	
- Linux:
	- `A` - движение влево
	- `D` - движение вправо
	- `S` - остановиться
	- `Space` - прыжок
	- `Q` - выход из программы


## 📝 Задания для самостоятельной работы

1. Изучить наследование на примере построения диаграмм иерархий для классов: 
	ConsoleEnemy, ConsoleShip.  
   
2. Изучить создание и хранение объектов игры на примере классов: 
	ConsoleUIFactory, Game.  

3. Создать:
	- летающего врага;
	- прыгающего на месте врага;
	- передвигающуюся платформу, на которую марио может запрыгнуть и переместиться 
		по карте над морем.

4. Изучить архитектуру построения уровней. Сделать новый второй уровень 
	и создать третий уровень.

5. Внимательно изучить весь код и подготовить ответы на вопросы.

6. Изучить, как построена кроссплатформенность в приложении. 

На данный момент кроссплатформенность не является серьёзной проблемой и для всего 
существуют кросплатформенные библиотеки, но сам принцип построения архитектуры 
приложений продолжает оставаться актуальным.  

7. 🎯 Создать уровень, который не сможет пройти преподаватель.
